CNPJ alfanumérico 2026: o novo CNPJ 2026 e mudanças no CNPJ

Novo formato de CNPJ chegando em 2026: letras entram na jogada e os sistemas precisam correr para não quebrar.

Publicado em 12 nov 2024 Por Fernando Bade Leitura de 8 minutos

Se você achava que o CNPJ brasileiro já estava “definido”, prepare-se para uma mudança inesperada. A Receita Federal do Brasil publicou a Instrução Normativa RFB nº 2.229/2024 que institui o novo CNPJ alfanumérico, em vigor a partir de julho de 2026. Serviços e Informações do Brasil.

Para quem gerencia produto, desenvolve sistemas ou cuida para que nada quebre no back-end, entender essa transição evita dor de cabeça. Neste artigo explico o que muda, por que foi criado, como funciona o cálculo de dígitos verificadores (sim, letras agora contam) e o que os desenvolvedores precisam fazer.

Quer experimentar agora? Teste o validador gratuito de CNPJ e veja como o algoritmo reage aos formatos atuais.

Por que o CNPJ vai ter letras?

Escassez de números disponíveis

O formato atual do CNPJ é composto por 14 dígitos exclusivamente numéricos. Esse padrão funcionou bem até aqui, mas a demanda por novas inscrições — filiais, MEIs, startups, franquias — cresceu tanto que o número de combinações possíveis está próximo do limite. Capitalist e Gazeta Mercantil destacam essa corrida por novos números.

Solução = letras + números

Para dar mais fôlego ao cadastro nacional, a Receita decidiu que a partir de julho de 2026 as novas inscrições terão formato alfanumérico, combinando letras maiúsculas e números. Apenas CNPJs emitidos após a virada passam a receber caracteres alfanuméricos, preservando todas as inscrições existentes. JusBrasil reforça que ninguém precisará trocar o número atual.

Benefícios esperados

  • Aumento massivo da quantidade de combinações possíveis, prolongando a vida útil do cadastro. Tecnoblog explica que o salto é exponencial.
  • Modernização do cadastro empresarial brasileiro, alinhando-se a padrões internacionais.
  • Menor risco de “acabarem” os números e travarem aberturas de novas empresas.
  • Sistemas públicos e privados obrigados a rever validações rígidas, tornando-se mais flexíveis. CFC já alerta para atualizações de cadastros.

O que muda no formato do CNPJ

Estrutura antiga vs. nova

O modelo numérico permanece reconhecido, mas o novo formato combina letras e números nas 12 primeiras posições e mantém dois dígitos verificadores numéricos ao final. Veja a comparação direta:

  • Formato antigo: 12.345.678/0001-90 — 14 dígitos numéricos.
  • Formato novo: segue com 14 caracteres, mas as 12 primeiras posições são alfanuméricas e os dois últimos dígitos continuam numéricos.

Distribuição prática das posições:

  1. Primeiras 8 posições: raiz da empresa com letras ou números. Portal Contábeis já lista casos ilustrativos.
  2. Quatro posições seguintes: identificação do estabelecimento, também alfanumérica. CRCPR.
  3. Últimos dois dígitos: continuam numéricos e cumprem a função de dígitos verificadores. Serviços e Informações do Brasil.

Exemplo ilustrativo: A1B2C3D4/E5F6-78 (mantendo barra e hífen na máscara padrão). Contador Direto já usa exemplos similares para treinamentos.

O que não muda

  • Os CNPJs já existentes continuam válidos e seguem o formato antigo. Portal Contábeis.
  • A estrutura segue com 14 caracteres.
  • A lógica de cálculo do dígito verificador via módulo 11 permanece, apenas adaptando as letras para valores numéricos. Serviços e Informações do Brasil.

Validação e dígito verificador

Desenvolvedor, é aqui que sua vida pode complicar se você não se antecipar. Cada caractere alfanumérico precisa virar um número antes de participar do cálculo do dígito verificador.

Conversão de letras para valores

Cada caractere do novo CNPJ (letra ou número) deve ser convertido para um valor numérico por meio do código ASCII menos 48. Assim, A (65) vira 17 e Z (90) vira 42. Os dígitos de 0 a 9 mantêm os valores tradicionais. Serviços e Informações do Brasil documenta a regra completa.

Cálculo módulo 11 adaptado

Depois de converter letras e números, aplica-se o módulo 11 com a mesma lógica tradicional: pesos 5-2 para o primeiro dígito, pesos 6-2 (com o primeiro DV) para o segundo. Se o resto for menor que 2, o dígito vira 0; caso contrário, usa-se 11 menos o resto. CNPJ Dados e Nova Regra do CNPJ já explicam essa adaptação.

Ver código JavaScript simplificado para validar CNPJ alfanumérico 
// --------------------------------------------
// Validador de CNPJ Alfanumérico (versão simplificada)
// Compatível com o novo formato de CNPJ a partir de 2026
// --------------------------------------------

function valorPorCaractere(ch) {
  const code = ch.charCodeAt(0);

  if (code >= 48 && code <= 57) {
    return code - 48;
  }

  if (code >= 65 && code <= 90) {
    return code - 48;
  }

  throw new Error(`Caractere inválido no CNPJ: ${ch}`);
}

function calcularDVs(cnpj12) {
  const valores = cnpj12.split("").map(valorPorCaractere);
  const pesos1 = [5, 4, 3, 2, 9, 8, 7, 6, 5, 4, 3, 2];
  let soma1 = 0;

  for (let i = 0; i < 12; i++) {
    soma1 += valores[i] * pesos1[i];
  }

  const resto1 = soma1 % 11;
  const dv1 = resto1 < 2 ? 0 : 11 - resto1;

  const valores13 = [...valores, dv1];
  const pesos2 = [6, 5, 4, 3, 2, 9, 8, 7, 6, 5, 4, 3, 2];
  let soma2 = 0;

  for (let i = 0; i < 13; i++) {
    soma2 += valores13[i] * pesos2[i];
  }

  const resto2 = soma2 % 11;
  const dv2 = resto2 < 2 ? 0 : 11 - resto2;

  return [dv1, dv2];
}

function validarCNPJ(cnpj) {
  const limpo = cnpj.replace(/[^A-Z0-9]/gi, "").toUpperCase();

  if (limpo.length !== 14) return false;

  const parte12 = limpo.slice(0, 12);
  const dvInformado1 = Number(limpo.charAt(12));
  const dvInformado2 = Number(limpo.charAt(13));

  const [dv1, dv2] = calcularDVs(parte12);

  return dv1 === dvInformado1 && dv2 === dvInformado2;
}

console.log(validarCNPJ("A1B2C3D4E5F6-78".replace(/-/g, "")));
// Retorna true ou false dependendo se o CNPJ é válido

Esse script serve como guia rápido. Para produção, utilize bibliotecas testadas e mantenha logs de auditoria.

Impactos para sistemas e desenvolvedores

Expressões regulares, bancos de dados e integrações precisarão ser atualizados. Validações rígidas para apenas números irão falhar com CNPJs alfanuméricos.

  • Regexs precisam aceitar letras nas 12 primeiras posições, algo como ^[A-Z0-9]{12}[0-9]{2}$. Nova Regra do CNPJ sugere padrões similares.
  • Colunas armazenadas como número devem virar texto. Bigints não aceitam caracteres.
  • Bibliotecas de validação precisam suportar ASCII e módulo 11 adaptado. GitHub já reúne códigos de referência.
  • Teste antecipado com CNPJs fictícios evita falhas em produção.

Impactos para empresas e times de produto

Empresas com CNPJ vigente

Nenhuma inscrição existente será alterada. Mas sistemas internos precisam ser auditados para garantir que aceitarão novos parceiros com letras no identificador. Atualize cadastros, ERPs e integrações para não travar vendas ou faturamento.

Novos estabelecimentos a partir de 2026

Toda inscrição emitida após julho de 2026 pode nascer alfanumérica. Atualize nota fiscal, e-commerce, cadastro de fornecedores e fornecedores de pagamento. Gazeta Mercantil lembra que a virada será gradual.

No próprio site da Receita existe uma página dedicada ao tema para empresas acompanharem a evolução. Serviços e Informações do Brasil reúne perguntas frequentes.

Para desenvolvedores (esse é seu playground)

  • Reveja expressões regulares e validações server-side e client-side.
  • Garanta que colunas e variáveis não forcem parse numérico.
  • Faça testes automatizados com casos alfanuméricos.
  • Use o repositório gerador-validador-cnpj para experimentar com dados e ferramentas.
  • Documente a mudança no backlog e comunique stakeholders.

Curiosidades que poucos contam

  • A mudança lembra a transição do IPv4 para o IPv6: de combinações finitas para um espaço quase infinito. Tecnoblog traça esse paralelo.
  • Com letras nas 12 primeiras posições, o volume de combinações explode. Espaço Legislação TOTVS calcula o ganho.
  • A Receita afirmou que a transição será progressiva e acompanhada de guias oficiais. Serviços e Informações do Brasil.
  • Há projetos no GitHub que já implementam calculadoras de DV para CNPJ alfanumérico, mesmo antes da virada.

Conclusão

A introdução do CNPJ alfanumérico em julho de 2026 é uma mudança técnica e estratégica. Os números atuais permanecem válidos, mas os sistemas precisam aceitar letras onde antes só existiam dígitos. Antecipe-se, previna falhas e mantenha seu produto robusto.
